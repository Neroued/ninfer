# MTP M1 W8G32 Linear Operator Family Implementation Plan

**Goal:** Land MTP roadmap milestone M1 / requirement R-L1-1 by making
`kernels::linear()` correctly execute W8G32 row-split weights for the five MTP dense/fused linear
families across `T == 1`, `2 <= T <= 16`, and `T > 16`, with numerical verification and a committed
baseline bench report.

**Requirements:** `docs/2026-07-03-mtp-roadmap.md` M1 and
`docs/2026-07-03-mtp-implementation-requirements.md` §3 / R-L1-1. The W8 metadata and payload math
come from `docs/q5090_packed_file_format_v3.md` and `tools/q5090_convert/{qtypes,packing}.py`; the
older W8G128 discussion in `docs/2026-07-02-mtp-foundation-part2-operators.md` §5 is guidance only
after replacing every group/payload parameter with W8G32.

**Execution Mode:** Main agent direct implementation. Subagent-driven implementation is intentionally
not used because the user explicitly required direct implementation, the change is tightly coupled
through the linear codec/dispatch/test harness, and splitting implementation ownership would create
shared-file contention in `linear_plan`, `linear.cpp`, and `test_linear.cpp`. Subagents are used only
for the two required reviews: one plan audit before implementation and one code review after self-test.
Both review subagents must use the current strongest available model; weaker, cheaper, legacy, or
compatibility-oriented models are not allowed.

**Architecture:** Add W8G32 as a first-class `LinearFormat` and codec. Reuse the existing
codec-templated generic GEMV and multi-step GEMM launchers after making the small-T launcher handle
32-wide groups. For M1 LargeT, route W8G32 to the correct generic multi-step GEMM path instead of the
existing tensor-core MMA kernel, because the current MMA path is K64-specific in group count, scale
indexing, and shared staging. This satisfies the M1 "correct generic GEMM" option without touching
Q4/Q5/Q6 MMA behavior.

## Non-Goals

- No W8 fused epilogue, direct compact multi-output writes, W8 gateup+SiLU, residual fusion, or hot
  shape tuning; these stay in M5.
- No model card changes, MTP weight binding, `mtp_forward`, MTP scheduling, GDN/KV/verify round work,
  or runtime MTP semantics; these stay in M2/M3.
- No q5090 format, converter, parser, or artifact schema changes; M0 already fixed W8G32 format work.
- No behavior changes to existing Q4/Q5/Q6 codec math, policy choices, or numerical tolerances.
- No new tests that preserve legacy unsupported-W8 behavior, assert source structure, or only exist to
  inflate coverage.

## Scope And Ownership

- `src/kernels/linear/codec/linear_codec.cuh`: add a W8G32 codec with `kGroupK = 32`, 32 signed int8
  code bytes per group, no high plane, one FP16 scale per group, and `dequant = int8 * scale`.
- `src/kernels/linear/linear.cpp`: accept W8G32 in validation, with q5090 row-split metadata checks
  for group 32, FP16 scales, no high plane, `K_pad = align_up(K,128)`, non-null code/scale planes, and
  correct payload minimum. Keep dense validation and Q4/Q5/Q6 validation behavior unchanged.
- `src/kernels/linear/plan/linear_plan.{h,cpp}`: add `W8G32_RowSplit`; register the five MTP W8
  family keys as two new dimensions (`[5120,10240]`, `[14336,5120]`) plus the existing
  `[5120,6144]`, `[34816,5120]`, and `[5120,17408]` dimensions paired with W8G32 format; route T1 to
  generic GEMV and both SmallT/LargeT to the generic multi-step GEMM policy.
- `src/kernels/linear/reference/linear_generic_lowbit_gemv.cu`: dispatch W8G32 T1 to the new codec.
- `src/kernels/linear/gemv/linear_rowsplit_gemm_multistep.{cuh,cu}`: dispatch W8G32 and guard lanes
  above `Codec::kGroupK / 2` so the same kernel is correct for 32-wide and 64-wide groups.
- `src/kernels/linear/gemm/linear_rowsplit_gemm_mma.{cuh,cu}`: do not change Q4/Q5/Q6 MMA behavior in
  M1; W8G32 LargeT does not dispatch here.
- `tests/kernels/test_linear.cpp`: delete the unsupported-W8 behavior test and replace it with W8G32
  numerical and metadata coverage that follows `docs/l1-op-test-standard.md`.
- `bench/linear_op_bench.cu`: register the five W8G32 MTP shape targets so `--all-targets` includes
  the M1 W8 baseline rows, while preserving W8G32 group/plane descriptor construction from M0.
- `docs/bench/`: add a short W8G32 baseline report with the `--t-sweep 1,2,3,5,6,16,64` data.

## Coordination Points

- `LinearFormat` changes must land with launcher switch coverage, or the default dispatch path will
  silently do nothing or throw from the wrong layer.
- The generic multi-step kernel currently assumes all 32 lanes own useful pairs for a 64-wide group.
  W8G32 requires an explicit `lane < Codec::kGroupK / 2` guard; this is the highest-risk shared
  kernel edit because it also compiles Q4/Q5/Q6.
- The five W8 family registrations share `ShapeFamily` with existing text shapes for three
  dimensions. Dispatch correctness depends on the `(format, shape, regime)` key, not on duplicating
  shape enum values by qtype.
- `linear_op_bench --all-targets` will produce additional W8 rows after M1. The report must identify
  the W8 subset used for the M1 baseline.

**Reading List:**

**Implementation:**
- Add `W8G32Codec`:
  - `kBits = 8`;
  - `kGroupK = 32`;
  - `kBytesPerRowPerGroup = 32`;
  - `load_group()` reads signed `int8` codes and applies the FP16 scale for one 32-code group;
  - `load_pair()` returns one pair for lanes `0..15`; lanes `16..31` are never consumed by the
    guarded small-T kernel.
- Add W8G32 cases to generic GEMV and multi-step GEMM launch switches.
- In `linear_rowsplit_gemm_multistep_kernel`, skip lanes `>= Codec::kGroupK / 2` before reading codes
  or activation pairs. Q4/Q5/Q6 keep all lanes active because their `kGroupK / 2` is 32.

**Definition Of Done:**
- The W8 codec compiles without reusing K64 high-plane logic.
- Q4/Q5/Q6 launcher behavior is unchanged except for the no-op lane guard that is true for all lanes
  on K64 codecs.

### Task 3: Validation, Format Classification, And Plan Policy

**Reading List:**
- `include/qus/kernels/linear.h`.
- `src/kernels/linear/linear.cpp`.
- `src/kernels/linear/plan/linear_plan.h`.

- `docs/m3-linear-backend-framework.md` §5, §6, §10.
- `docs/q5090_packed_file_format_v3.md` §9.2.

**Implementation:**
- Generalize row-split metadata validation to accept an explicit group size, base bytes per group, and
  high bytes per group.
- Add a W8G32 validation branch:
  - `qtype == QType::W8G32_F16S`;
  - `layout == QuantLayout::RowSplit`;
  - `group == group_size == 32`;
  - `q5090_scale_dtype == ScaleDType::FP16`;
  - `qhigh == nullptr`;
  - `high_plane_bytes == 0`;
  - `padded_shape[0] == shape[0]`;
  - `padded_shape[1] == align_up(shape[1], 128)`;
  - `qdata` and `scales` non-null for every valid descriptor, including empty T;
  - payload is at least `align_up(N * (K_pad / 32) * 32, 256) + N * (K_pad / 32) * 2`.
- Add `LinearFormat::W8G32_RowSplit`, names, and classification.
- Add the new W8 dimensions to `ShapeFamily` classification:
  - `MtpFc5120x10240`;
  - `MtpAttnIn14336x5120`.
- Keep the existing `Out5120x6144`, `MlpGateUp34816x5120`, and `MlpDown5120x17408` families and
  register them for W8G32 through the format axis.
- Resolve W8G32 plans:
  - `T1` -> `GenericLowbitGemv`;
  - `SmallT` -> `RowsplitLowbitGemmMultistep`;
  - `LargeT` -> `RowsplitLowbitGemmMultistep` as the M1 correct generic GEMM fallback with
    `uses_tensor_cores = false`.
- Keep the existing `k % 8` MMA fallback rule scoped to MMA-routed formats; W8G32 does not need a
  separate LargeT fallback because it never selects the K64 MMA kernel.

**Definition Of Done:**
- `linear()` accepts valid W8G32 descriptors and rejects malformed W8G32 descriptors.
- All three regimes produce a concrete plan for each of the five MTP W8 family keys.
- Existing Q4/Q5/Q6 registered T1 plans and LargeT MMA plans remain selected as before.

  - five MTP family shapes `[5120,10240]`, `[14336,5120]`, `[5120,6144]`, `[34816,5120]`,

  tensor-core MMA path; do not define custom tolerance values.

  - bad payload size rejects;

  - empty T with valid descriptor succeeds with null x/out;
  - bad group size, non-null high plane, nonzero high-plane bytes, wrong padded K, and wrong scale
    dtype reject.

**Definition Of Done:**
- `qus_linear_test` proves W8G32 correctness for T1, SmallT, and LargeT on synthetic and five real

# M2.5 — Hardening & Cleanup — Plan (Codex-self-contained)

> **Current status:** many M2.5 hardening tasks have landed in code. In particular, current
> structural checks cover graph-readiness fixes, EOS handling, `FileTap`, and cleanup invariants.
> This plan is retained as a hardening DoD and verification reference; environment-heavy gates
> should be treated as verify/keep-current unless freshly rerun.

> Self-contained for Codex. Reuses the execution machinery in
> [`docs/plans/l1-tier1-simple-ops.md`](l1-tier1-simple-ops.md) (subagent workflow + prompt templates
> + ncu procedure) and the frozen test framework
> ([`docs/l1-op-test-standard.md`](../l1-op-test-standard.md),
> [`tests/kernels/op_check.h`](../../tests/kernels/op_check.h)).

**Goal:** Fix the quality / process / test-rigor issues found in review, in priority order, **without
changing op behavior** (except removing dead/over-built code). Correctness stays green throughout.

**Why (historical):** L1 + M2 were functionally correct, but review found quality / process /
test-rigor issues: dense `linear` was over-built, some gates lacked decode/chunked/unaligned/stress
coverage, graph-readiness invariants needed fixing, EOS + parity `Tap` were absent, and cleanup
inconsistencies remained. Current code has landed many of these fixes.

**Tech Stack:** C++20, CUDA 13.1 (sm_120), CMake ≥ 3.28, gcc 13.3, build dir `build/`.

---

## Hard rules
- **Behavior-preserving** except deletions of dead/over-built code. All existing tests must stay green
  (`ctest --test-dir build --output-on-failure`); add the missing cases per task.
- The frozen test-framework **logic** stays read-only. Two scoped doc/API reconciliations are allowed
  and called out: (h-4) add a `WorkspaceArena&` param to `gated_delta_rule_recurrent` and update the
  catalog/design; (h-6) record the already-added presets in the standard's "frozen" list.
- **FORMAT before every commit:** `clang-format -i <changed files>` then `clang-format --dry-run
  --Werror <…>` (exit 0). Work on `master`, one commit per task.

## Tasks (priority order)

### Task h-1 — trim the over-built dense `linear` path  [High]
- **Reading:** `src/kernels/kernel/linear_dense.cuh`, `src/kernels/launcher/linear_dense.cu`,
  `docs/plans/l1-tier3-linear-attention.md` (Performance policy), `tests/kernels/test_linear.cpp` (dense cases).
- **Do:** the dense path feeds only `in_a`/`in_b` (`[48,5120]` BF16) — it must be **correct + simple**, not
  tensor-core-tuned. Delete the unreachable / over-tuned kernels and their dispatch arms + helpers:
  `linear_dense_gemm_bf16_full_kernel` (cp.async double-buffered, needs `N%64==0`),
  `linear_dense_gemm_bf16_half_wide_full_kernel` (needs `N%64==0`), the `dense_cp_async_*` helpers, and
  the unused `kDenseGemvWarps`. Collapse the GEMM dispatch to a **single** general masked GEMM
  (`linear_dense_gemm_kernel`); keep `linear_dense_gemm_tf32_kernel` **only** if the `FP32_CTRL` GEMM
  consistency test needs the precision, otherwise drop it too. Keep both GEMV kernels.
- **DoD:** `qus_linear_test` (all dense + quant cases) still PASS; sanitizer clean; `linear_dense.cuh`
  drops to roughly a GEMV + one GEMM (~300 lines, no unreachable kernels); clang-format clean.
- **Commit:** `refactor(kernels): trim over-built dense linear to a simple correct path`.

### Task h-2 — close `linear` test gaps  [High]

- **Do:** add to `test_linear.cpp` (frozen `Tolerance::linear_bf16()`): (a) a **stress** case per qtype

### Task h-3 — close GDN + block-parity gaps  [High]

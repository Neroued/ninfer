# M2.8 P7/P8 Workspace Lifetime and M3 Readiness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement
> this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace step-level workspace lifetime with block-scoped mixer/MLP rewind, then produce the
official M3 entry evidence package without starting kernel optimization.

**Architecture:** Execute P7 and P8 as one sequential plan. First change only workspace temporary lifetime,
prove the new policy against the shared short case from the P6 `step_reset` before-baseline, run the first
`long_2k` observation only after block-scoped workspace rewind is active, and commit the P7 smoke summary.
Then run the official M3 gate baseline under the new policy, commit the M3 gate summary, and write
`docs/m3-readiness.md`.

**Tech Stack:** C++ runtime/model code, existing `DeviceArena::mark()/rewind()`, existing `qus_e2e_bench`,
P4 tokenizer/decode tooling, P5 report validation/comparison/summary tooling, real q5090 weights, local HF
tokenizer files, JSON, Python 3 standard library.

## Scope

This merged plan covers both original index items:

- P7: workspace lifetime hardening from `step_reset` to `block_scoped_mixer_mlp_rewind`;
- P8: official M3 gate baseline, committed summary, and `docs/m3-readiness.md`.

Start this plan only after P6 is verified and the workspace is clean. P6 provides the required short
`step_reset` before-baseline for P7. P6 does not provide a `long_2k` before-baseline because the current
`step_reset` runtime accumulates workspace across layers and cannot complete the current long fixture within
the device workspace budget. The first valid `long_2k` observation is therefore owned by P7 after
block-scoped workspace rewind is implemented.

This plan owns:

- block-scoped workspace mark/rewind around every mixer and MLP block in `Qwen3_6_27B::run_layers`;
- report policy change from `step_reset` to `block_scoped_mixer_mlp_rewind`;
- structural and report tests for the workspace lifetime policy;
- local P7 after-workspace smoke report and local P7-vs-P6 comparison result;
- first real-weight `long_2k` observation under `block_scoped_mixer_mlp_rewind`;
- committed P7 smoke summary under `docs/bench/baselines/`;
- official P8 M3 gate raw report under local `profiles/e2e/`;
- decoded sidecars for the official gate report;

- `docs/m3-readiness.md`;
- P7/P8 status closeout in `docs/plans/m2.8-implementation-plan-index.md`.

This plan does not own:

- kernel optimization;
- kernel fusion;

- waiving any M2.8 standard requirement.

## Policy Invariants

The workspace policy after Task 2 is:

```text
workspace_lifetime_policy = "block_scoped_mixer_mlp_rewind"
```

Required runtime order:

```text
mark before attn_mix/gdn_mix
run mixer
run AfterMixer tap if enabled
rewind to mixer mark

mark before mlp_tail
run MLP
run AfterMlp tap if enabled
rewind to MLP mark
```

The policy applies to prefill and decode. Taps must stay before the corresponding rewind. `x` must be
allocated before the block mark and remain valid after rewind. Temporaries allocated inside the block must
not escape the block.

```cpp
int expect_order_after(const std::string& text, const char* anchor, const char* first,
                       const char* second, const char* message) {
    const std::size_t anchor_pos = text.find(anchor);
    if (anchor_pos == std::string::npos) {
        std::cerr << message << ": missing anchor `" << anchor << "`\n";
        return 1;
    }
    const std::size_t first_pos = text.find(first, anchor_pos);
    const std::size_t second_pos = text.find(second, anchor_pos);
    if (first_pos != std::string::npos && second_pos != std::string::npos &&
        first_pos < second_pos) {
        return 0;
    }
    std::cerr << message << ": expected `" << first << "` before `" << second << "` after `"
              << anchor << "`\n";
    return 1;
}

std::string slice_between(const std::string& text, const char* begin, const char* end) {
    const std::size_t begin_pos = text.find(begin);
    if (begin_pos == std::string::npos) { return {}; }
    const std::size_t end_pos = text.find(end, begin_pos);
    if (end_pos == std::string::npos) { return text.substr(begin_pos); }
    return text.substr(begin_pos, end_pos - begin_pos);
}
```

- [ ] **Step 4: Rework `run_layers` to mark and rewind per block**

Replace `Qwen3_6_27B::run_layers` in `src/model/qwen3_6_27b.cpp` with:

```cpp
template <class Tap>
void Qwen3_6_27B::run_layers(Tensor& x, Phase ph, Tap& tap) {
    for (int layer = 0; layer < kCfg.n_layers; ++layer) {
        const std::size_t mixer_mark = work_.mark();
        if (ModelConfig::is_full(layer)) {
            const int fidx         = ModelConfig::full_idx(layer);
            const FullLayerW& full = full_.at(static_cast<std::size_t>(fidx));
            attn_mix(full, x, fidx, ph);
            if constexpr (Tap::enabled) { tap(TapId::AfterMixer, layer, ph, x, ctx_.stream); }
            work_.rewind(mixer_mark);

            const std::size_t mlp_mark = work_.mark();
            mlp_tail(full.post_attn_norm, full.mlp, x, ph);
            if constexpr (Tap::enabled) { tap(TapId::AfterMlp, layer, ph, x, ctx_.stream); }
            work_.rewind(mlp_mark);
        } else {
            const int gidx       = ModelConfig::gdn_idx(layer);
            const GdnLayerW& gdn = gdn_.at(static_cast<std::size_t>(gidx));
            gdn_mix(gdn, x, gidx, ph);
            if constexpr (Tap::enabled) { tap(TapId::AfterMixer, layer, ph, x, ctx_.stream); }
            work_.rewind(mixer_mark);

            const std::size_t mlp_mark = work_.mark();
            mlp_tail(gdn.post_attn_norm, gdn.mlp, x, ph);
            if constexpr (Tap::enabled) { tap(TapId::AfterMlp, layer, ph, x, ctx_.stream); }
            work_.rewind(mlp_mark);
        }
    }
}
```

Do not move `AfterMixer` or `AfterMlp` taps after the rewind. Do not add synchronization; same-stream
kernel ordering is the required lifetime guarantee.
